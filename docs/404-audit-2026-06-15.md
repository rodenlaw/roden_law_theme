# Roden Law — 404 / Broken-URL Audit (2026-06-15)

**Trigger:** Analytics showing high page views to 404 pages.
**Method:** Live HTTP status check of all 2,211 known URLs (crawl export `redirects.csv` + `wp-content-inventory-urls.txt`), redirect-chain resolution, Semrush organic + backlink overlay, sitemap & homepage internal-link inspection. No redirect-following = original status; following = final landing status.

## Headline numbers
- **2,203 URLs checked** → 1,192 live (200) · 961 redirects (301) · **58 hard 404s**
- **37 redirects that 301 to a 404** (the redirect exists but its *target* is dead)
- **1 sitewide dead link in the primary header navigation** ← most likely driver of real-user 404 views
- Sitemap is **clean** (advertises 1,199 URLs, none of them dead) — so the 404 traffic is from Google's historical index, external backlinks, and internal links, **not** the sitemap.
- Site (WP Engine) rate-limited the crawl at ~16 concurrent (429s). Not a user-facing bug, but aggressive bots can get throttled.

---

## P0 — Sitewide dead link in header nav (fix first)
The Practice Areas dropdown (present on **every page**) contains:
```html
<li id="menu-item-3127" ...><a href="https://rodenlaw.com/?page_id=3126">View All Service Areas</a></li>
```
`?page_id=3126` returns **404** (page deleted; nav menu item still references its old page_id).
Every visitor who opens the Practice Areas menu and clicks "View All Service Areas" lands on a 404.
**Fix:** In WP Admin → Appearance → Menus, repoint menu item 3127 to the live `/practice-areas/` hub (200), or delete the item. (Added in commit `94f34fe fix(nav): add Practice Areas dropdown to header`.)

## P1 — Redirects that land on a 404 (37) — redirect target is dead
The redirect rule fires (301) but points at a URL that no longer exists.

### Pattern A — `/case-result/<slug>/` → `/case-results/<slug>/` → 404 (12)
Redirect pluralizes `case-result` → `case-results`, but the pluralized target slug doesn't exist (case-result posts deleted/renamed). Examples:
- `/case-result/950000-recovery-truck-accidents/` → `/case-results/950000-recovery-truck-accidents/` (404)
- `/case-result/600000-settlement-truck-accidents/` → … (404)
- `/case-result/200000-policy-limit-auto-accident/` → … (404)
(Full list in audit data.)

### Pattern B — `/blog-<slug>/` (legacy permalink) → `/blog/<slug>/` → 404 (25, incl. doubled `/blog/blog-<slug>/`)
Old hyphen-style blog permalinks 301 to the `/blog/` subdir, but the destination post was deleted. Several have **external backlinks** (real lost link equity), notably:
- `/blog-why-should-i-hire-an-accident-lawyer-after-an-accident/` → 404 — **7 backlinks / 5 domains** (Semrush)
- `/blog-why-is-having-a-personal-injury-lawyer-important/` → 404
- `/blog-i-was-in-a-car-accident-should-i-pursue-a-personal-injury-claim/` → 404

**Fix:** For each, either (a) restore/republish the post, or (b) repoint the redirect target to the closest live page (relevant practice-area or blog hub). Don't leave a 301→404 chain.

## P2 — Hard 404s with no redirect (58)
Old URLs that were live as recently as April 2026 (in the crawl export) and now 404 with no redirect.

**Old top-level practice-area pages** (moved under `/practice-areas/…`, never redirected):
- `/burn-injury-lawyers/`, `/construction-accident-lawyers/`, `/product-liability-lawyers/`, `/slip-and-fall-lawyers/`, `/workers-compensation-lawyers/`, `/wrongful-death-lawyers/`
- `/free-case-review/` (old CTA landing page)
- `/practice-areas/brunswick/`, `/practice-areas/general/`

**Deleted/renamed blog posts (19)** — e.g. `/blog/charleston-port-drayage-truck-accidents/`, `/blog/south-carolina-golf-cart-laws/`, `/blog/highway-hypnosis-what-it-is-why-it-is-a-crash-risk/` (has a backlink), `/blog/myrtle-beach-golf-cart-laws/`, `/blog/south-carolina-car-accident-settlement-amounts/`.

**Taxonomy / category archives:** `/category/lost-wages-support/`, `/class-action-category/dangerous-drugs/`, `/class-action-category/defective-medical-devices/`, `/class-action-category/defective-products/`, `/class-action-lawyers/dangerous-drugs/`.

**Raw WordPress permalinks (deleted posts):** ~20 URLs of form `?page_id=NNNN` and `?post_type=practice-area&p=NNNN`. These resolve to 404 because the underlying post was deleted; they linger in Google's index and old internal references.

**Content bug:** `/blog/request-your-free-case-review-today-main_phone_number/` — a template placeholder (`main_phone_number`) leaked into a published post slug.

**Fix:** Add 301s mapping each old URL to its closest live equivalent (e.g. `/burn-injury-lawyers/` → `/practice-areas/burn-injury-lawyers/`; deleted city-specific blog posts → relevant practice-area or location page). Where the placeholder post exists, fix/unpublish it.

## Root cause
The site went through a URL restructure (top-level `/x-lawyers/` → `/practice-areas/x-lawyers/`; `/who-we-are/attorneys/` → `/attorneys/`; city PI pages → `/practice-areas/.../{city}-ga|sc/`). The bulk of redirects (961) work, but three gaps remain: (1) a batch of old pages got no redirect, (2) some redirects point at targets that were themselves later deleted/renamed, (3) the header nav still references a deleted page by page_id.

## Recommended order of operations
1. **P0 nav fix** (1 menu item) — stops the largest stream of real-user 404 views immediately.
2. **P1 redirect-target fixes** (37) — repoint 301→404 chains to live pages; preserves backlink equity.
3. **P2 missing redirects** (58) — bulk-add 301s via the redirect plugin / .htaccess.
4. Re-crawl to confirm 0 hard 404s and 0 redirect→404 chains; submit updated coverage in GSC.
